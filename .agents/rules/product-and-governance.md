# Reglas de Gobernanza, UI y Documentación

## 1. Regla de Consistencia de Idioma
Definir un idioma oficial para la interfaz y documentación. Los textos visibles en la UI no deben mezclar idiomas (inglés y español) dentro de la misma pantalla para conservar la consistencia de producto.

## 2. Regla de Naming de Dominio
Conservar el naming de negocio actual que ya es útil (`income`, `outcome`, `business_type`), pero seguir la guía de convenciones unificada para nuevas variables de UI y mensajes de sistema.

## 3. Regla de Documentación Viva
Cuando el archivo README del repositorio declare una estructura o proceso obligatorio (como el uso de directorios de agentes), el repositorio debe cumplirlo estrictamente o, en su defecto, documentar una excepción explícita.
